着色器模块

概要

这一章节我们将完成渲染管线中可编程阶段的构建。目前我们只简单地考虑构建顶点着色器和片段着色器阶段。

与之前的图形API不同，Vulkan使用的着色器代码需要是SPIR-V格式的，它的优点是相比于易于我们阅读的GLSL和HLSL代码，SPIR-V代码更容易被转化为GPU使用的机器码，并且GLSL和HLSL代码可能会因为GPU产商所设定的不同标准产生在不同机器上的不兼容现象，SPIR-V代码就不会出现这种问题。

不过幸运的是，我们并不需要自己手写SPIR-V代码，这里我们将使用Vulkan SDK自带的glslc.exe来将GLSL代码编译为SPIR-V代码。

创建着色器模块主要分为四步：

  • 编写着色器代码
  • 将着色器文件编译为SPIR-V代码
  • 加载着色器文件到程序中并包裹在VkShaderModule对象中
  • 填写渲染管线着色器阶段信息

本文对应Vulkan Tutorial的Shader modules章节。

首先需要解释的是pName形参，该形参指定着色器代码的入口点。一般来讲我们的glsl着色器代码入口点为main函数，但是Vulkan允许我们指定其它函数名作为入口函数，这也意味着我们可以在一个着色器模块中绑定多个具有不同入口名的着色器文件。

另外，pSpecializationInfo指定的是需要传给着色器的特化常量的信息。特化常量是在着色器编译时指定的常量值。它们在着色器代码中声明为特化常量，并在编译时被替换为具体的值。这使得可以在不同的管线实例中使用不同的常量值，提高了性能，因为编译器能够针对这些常量进行优化。在 GLSL 中，特化常量通常使用 const 关键字来声明，但在 Vulkan 中，需要使用特化常量的机制来定义和传递这些值。特化常量与C中的预编译命令 #define 类似，不过特化常量是有类型的，编译器能够进行类型检查，并且特化常量的作用域通常局限于声明它们的着色器阶段，因此相比来讲特化常量具有类型安全和更好的作用域控制。
