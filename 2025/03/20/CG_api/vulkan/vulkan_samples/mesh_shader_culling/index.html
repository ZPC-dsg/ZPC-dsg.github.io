Mesh shader culling

前言

上一个示例中我们看到了Vulkan中mesh shader的基本使用方式。本示例中，我们将进一步探究基于mesh shader的管线，结合task shader和mesh shader实现基本的四边形剔除效果。我将在涉及到task shader和mesh shader相关代码的时候再来介绍相关的一些实现细节。事实上该示例的官方文档中给出了很多有关mesh shader的资料供参考，希望进一步深入理解mesh shader中有关细节的读者可以参阅该文档。

其中前两个常量代表task shader一个工作组在x、y方向上的维度大小，后两个则代表mesh shader一个工作组的x、y维度大小。下面的结构体会用作一个task shader工作组和这个工作组内的task shader调用的所有mesh shader工作组之间的共享变量。

回到task shader代码，接下来代码启动拓展GL_KHR_shader_subgroup_ballot，该拓展提供了对着色器子组投票和选票功能的支持。该扩展提供了一组函数，允许着色器中的线程在子组内进行投票。这些投票可以用来确定子组内的某些条件是否满足。后续在涉及到该拓展相关函数的时候再具体说明。

然后注意程序中的taskPayloadSharedEXT关键字，该关键字指定后续的变量将作为本组内的task shader以及它们所调用的mesh shader的共享变量，不过task shader对该变量可读可写，而mesh shader只能读取这个变量。另外，用taskPayloadSharedEXT修饰的变量只能有一个。

接下来来分析主函数逻辑。首先得到每个小方形大小，根据代码公式，显然程序是将整个大方形设置为2x2的，小方形的大小就取决于总的调用次数gl_NumWorkGroups.xy * gl_WorkGroupSize.xy，工作组大小在程序中是始终不变的（始终设置为头文件中对应的常量），但工作组维度是会根据UI界面的第三个选项栏的选择而改变的，后续在录制命令代码中也会看到。然后代码得到当前task shader调用处理的小方形在大方形中的偏移（相对于左上角，代码是直接设定该大矩形左上角的坐标就是(0，0)），由此就可以计算出小方形相对于摄像机的位置坐标（还记得程序中将剔除中心位置设置为摄像机x和z坐标的相反数吗？这里p0 = position + offset其实就是p0 = offset - camera.position.xz，程序初始时设置相机的x、z坐标均为1，于是此时所有小方形的p0就正好全在(-1，1)内，并且相机坐标就正好是这个大方形的中心坐标），然后再偏移一个小方形的大小就可以计算出该小方形的四个顶角相对于摄像机的位置接下来程序就可以通过判断这四个顶角是否都在提出半径内来决定该小方形的可见性，设置为只有四个顶角都在半径内才算可见。确定可见性之后，代码调用subgroupBallot函数来根据可见性这个布尔值进行投票，返回一个uvec4用于表示整个线程组的投票结果。之所以返回uvec4是因为一般一个task shader工作组最多支持128个线程，而一个uint有32位，4个uint就可以表示128位，从而存储128个线程的投票结果了。另外需要注意的是，这个投票函数是隐式同步的，意味着该函数的调用实际相当于隐式插入一个同步点，所有的线程都会在这里进行同步以保证后续所有线程都能获取到最后完整的投票结果。投票之后，代码将所有可见的小方形索引放入共享数据的offsets数组中，为了能够顺序并且不重叠地保存每个索引，代码使用subgroupBallotExclusiveBitCount函数获取投票结果中当前线程之前的（也就是线程ID小于当前线程）置位总数作为当前线程的索引值。最后，线程组中的第一个线程负责共享变量中的一些线程组整体变量的设置以及后续mesh shader的调用。由于position和size都是线程组内不变的，因此只需要一个线程设置一次就可以了。另外，对上一个示例的分析中也提到了EmitMeshTasksEXT一个线程组内必须调用且只需要调用一次就可以了，这里就让第一个线程负责调用，并且分配的mesh shader线程组数就是可见的小方形数以便每个mesh shader线程组负责一个小方形的顶点和图元信息处理。

接下来来看mesh shader的代码：

可以看到，每个可见的小方形的task shader会调用4个mesh shader，这个小方形又会均匀分为4个小方形供每个mesh shader分别处理。每个mesh shader的调用会生成小方形的4个顶点和两个三角形，因此整个mesh shader工作组的最大输出顶点数可以设置为4 * numMeshInvocationsX * numMeshInvocationsY，最大输出图元数可以设置为2 * numMeshInvocationsX * numMeshInvocationsY。然后可以看到每个着色器调用输出的仍然是每个顶点的颜色信息供后续片段着色器直接使用。

接下来来看主函数：对上一个示例的分析也提到了SetMeshOutputsEXT函数同样只需要在一个工作组的一次调用中指定就可以了（多个调用中指定的话需要指定的数值一致，不然会报编译期错误），这里直接就在第一个工作组内指定即可。然后每次调用从共享变量中获取到当前mesh shader工作组所处理的方形的整体偏移量，然后再获取到当前调用所处理的小方形在这个方形中的偏移量从而计算出每个小方形四个顶点的总体偏移量（相对于整个大方形的左上角，即原点），这样就可以通过在共享变量中存储的剔除中心位置计算出每个顶点相对于摄像机的位置，这个位置就作为最后的NDC坐标（位于NDC之外的会被后续光栅化阶段剔除掉），这个坐标需要通过gl_MeshVerticesEXT中的gl_Position来指定。另外，mesh shader还需要指定每个生成的图元中包含顶点的索引，该索引向量会保存到gl_PrimitiveTriangleIndicesEXT数组中。最后，指定每个顶点的输出颜色即可。需要注意每个mesh shader的输出变量都是整个线程组内的一个数组而不是每次调用输出一个数组或者别的数据结构。
