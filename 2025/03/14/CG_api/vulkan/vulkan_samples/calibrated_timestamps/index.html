Calibrated timestamps

前言

前一篇文章中我们看到了Vulkan GPU上的计时器——时间戳。时间戳的使用还是比较受限的，它只能计算单队列上一个完整render pass的耗时。本示例将引入一个更加精确且适用于任何代码块上（CPU、GPU都行）耗时测量的工具——校准时间戳（calibrated timestamps）。

本示例使用的场景渲染框架仍然是时间戳示例使用的hdr bloom框架。所以具体的渲染代码几乎没有任何变化，本文将不再放上这些代码，这部分读者可自行参考我的上一篇博客。这篇文章将仅仅把注意力集中在校准时间戳的准备和使用逻辑上。

get_device_time_domain

在准备阶段，程序会尝试查询设备支持的时间域并挑选出合适的那个。时间域（time domain）表示时间戳的来源时钟类型。Vulkan 允许不同硬件（如 CPU 和 GPU）使用独立的时钟源，而 time domain 用于指定要校准的时钟源类型。另外，最大偏差（max deviation）表示校准后的时间戳的最大可能误差范围（单位：纳秒）。它是一个保守估计值，表示校准后的时间戳与实际理想时间戳之间的最大偏差。get_device_time_domain函数代码如下：

可以看到对硬件支持的时间域的查询也是分两步进行的：首先查询支持的时间域数目，然后查询所有的支持时间域。然后对于每个时间域，填写关于它的VkCalibratedTimestampInfoEXT信息，这在后续创建校准时间戳的时候会用到。

get_device_time_domains函数接下来选取VK_TIME_DOMAIN_DEVICE_EXT枚举指定的时间域，它使用 GPU 设备内部时钟（如 GPU 计数器）。适用于测量 GPU 操作的耗时（如着色器执行时间）。实际上设备还可能支持的时间域有VK_TIME_DOMAIN_CLOCK_MONOTONIC_EXT、VK_TIME_DOMAIN_CLOCK_MONOTONIC_RAW_EXT和VK_TIME_DOMAIN_QUERY_PERFORMANCE_COUNTER_EXT，它们各自有不同的适用场景。本示例仅适用VK_TIME_DOMAIN_DEVICE_EXT指定的时间戳进行时间测量。
